<h1>Cert Checker Script</h1>

<p>Этот скрипт предназначен для проверки сертификатов и keystore файлов в секретах OpenShift проекта. Сценарий сканирует все секреты в проекте, извлекает и проверяет сертификаты и keystore файлы на срок их действия. Результаты проверки записываются в лог-файл.</p>

<h2>Установка</h2>
<ol>
    <li>Скачайте скрипт и поместите его в удобную для вас директорию.</li>
    <li>Убедитесь, что у вас установлен и настроен <code>oc</code> CLI для работы с OpenShift.</li>
    <li>Убедитесь, что у вас установлены утилиты <code>jq</code>, <code>keytool</code>, и <code>perl</code>.</li>
</ol>

<h2>Использование</h2>

<h3>Запуск скрипта</h3>
<ol>
    <li>Дайте права на выполнение скрипта:
        <pre><code>chmod +x cert_checker.sh</code></pre>
    </li>
    <li>Запустите скрипт:
        <pre><code>./cert_checker.sh</code></pre>
    </li>
</ol>

<h3>Параметры ввода</h3>
<ul>
    <li><strong>Лог будет сохранён в текущей папке, согласны? (y/n):</strong> 
        <ul>
            <li><code>y</code>: Логи будут сохранены в текущей директории.</li>
            <li><code>n</code>: Укажите полный путь до директории, где должны быть сохранены логи.</li>
        </ul>
    </li>
    <li><strong>Введи токен:</strong> Ваш токен для аутентификации в OpenShift.</li>
    <li><strong>Введи сервер:</strong> Адрес вашего OpenShift сервера.</li>
    <li><strong>Напиши название проекта:</strong> Название проекта в OpenShift, в котором будут проверяться секреты.</li>
    <li><strong>Введи дефолтный пароль сикретов твоей системы:</strong> Пароль для keystore файлов в вашей системе.</li>
</ul>

<h3>Описание работы скрипта</h3>
<ol>
    <li><strong>Создание и очистка директории логов:</strong>
        <ul>
            <li>Создается директория для логов (<code>cert_checker_logs</code>) в указанной пользователем директории.</li>
            <li>Очищается содержимое директории логов перед началом новой проверки.</li>
        </ul>
    </li>
    <li><strong>Аутентификация и выбор проекта:</strong>
        <ul>
            <li>Выполняется вход в OpenShift с использованием предоставленного токена и сервера.</li>
            <li>Пользователю предлагается ввести название проекта, который будет проверяться.</li>
        </ul>
    </li>
    <li><strong>Сбор и проверка секретов:</strong>
        <ul>
            <li>Собираются все секреты в указанном проекте.</li>
            <li>Проверяются файлы сертификатов и keystore файлов в каждом секрете.</li>
            <li>Если в секрете есть файлы сертификатов или keystore, они декодируются и проверяются с помощью <code>keytool</code>.</li>
        </ul>
    </li>
    <li><strong>Проверка сроков действия:</strong>
        <ul>
            <li>Вычисляется количество дней до окончания срока действия каждого сертификата или keystore файла.</li>
            <li>Результаты проверки записываются в лог-файл.</li>
        </ul>
    </li>
</ol>

<h3>Логирование</h3>
<p>Все результаты проверки записываются в файл <code>cert_check_log.txt</code> в директории логов. В лог-файл записываются следующие данные:</p>
<ul>
    <li>Название проекта.</li>
    <li>Название секрета и файла, который проверяется.</li>
    <li>Срок действия сертификата или keystore файла.</li>
    <li>Предупреждения о приближающемся окончании срока действия.</li>
</ul>

<h3>Выводы</h3>
<ul>
    <li><strong>Цепочки keystore:</strong> Список цепочек для каждого keystore файла.</li>
    <li><strong>Сроки действия сертификатов:</strong> Количество дней до окончания срока действия каждого сертификата или keystore файла.</li>
</ul>

<h2>Примечания</h2>
<ul>
    <li>Если пароль для keystore файла не подходит, пользователь может ввести новый пароль или пропустить проверку этого файла, введя <code>skip</code> или <code>s</code>.</li>
    <li>Если в секрете нет сертификатов или keystore файлов, это будет отображено в лог-файле.</li>
</ul>
